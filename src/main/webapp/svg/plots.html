1.1 Introduction

The ‘plot’ path operation can be used to append a line or curve to the path that goes through a large number of coordinates. The coordinates of the plot data are computed by evaluating a mathematical expression. Since the SVG Plot Module Extension comes with a mathematical engine, you can specify this expression and then have the desired coordinates produced for you, automatically.
Since this case is quite common when plotting a function, the syntax is easy: Following the ‘plot’ element and its attributes, you directly provide a 〈coordinate expression〉. It looks like a normal coordinate, but inside you may use a special macro, which is #x by default, but this can be changed using the ‘variable’ attribute. The 〈coordinate expression〉 is then evaluated for different values for #x and the resulting coordinates are plotted.

The SVG Plot Module Extension (conceptually) uses a two-stage process for generating plots. First, a plot stream must be produced. This stream consists (more or less) of a large number of coordinates. Second a plot handler is applied to the stream. The plot handler "does something" with the stream. The handler will issue line-to commands to the coordinates in the stream by default. However, using the ‘connected’ attribute, the handler can also issue appropriate curve-to commands in order to smooth the curve.

Categories:
Graphics element, shape element

Any number of the following elements, in any order:
  • animation elements — ‘animate’, ‘animateColor’, ‘animateMotion’, ‘animateTransform’, ‘set’

  • conditional processing attributes — ‘requiredFeatures’, ‘requiredExtensions’, ‘systemLanguage’

  • graphical event attributes — ‘onfocusin’, ‘onfocusout’, ‘onactivate’, ‘onclick’, ‘onmousedown’, ‘onmouseup’, ‘onmouseover’, ‘onmousemove’, ‘onmouseout’, ‘onload’
  • presentation attributes — ‘alignment-baseline’, ‘baseline-shift’, ‘clip’, ‘clip-path’, ‘clip-rule’, ‘color’, ‘color-interpolation’, ‘color-interpolation-filters’, ‘color-profile’, ‘color-rendering’, ‘cursor’, ‘direction’, ‘display’, ‘dominant-baseline’, ‘enable-background’, ‘fill’, ‘fill-opacity’, ‘fill-rule’, ‘filter’, ‘flood-color’, ‘flood-opacity’, ‘font-family’, ‘font-size’, ‘font-size-adjust’, ‘font-stretch’, ‘font-style’, ‘font-variant’, ‘font-weight’, ‘glyph-orientation-horizontal’, ‘glyph-orientation-vertical’, ‘image-rendering’, ‘kerning’, ‘letter-spacing’, ‘lighting-color’, ‘marker-end’, ‘marker-mid’, ‘marker-start’, ‘mask’, ‘opacity’, ‘overflow’, ‘pointer-events’, ‘shape-rendering’, ‘stop-color’, ‘stop-opacity’, ‘stroke’, ‘stroke-dasharray’, ‘stroke-dashoffset’, ‘stroke-linecap’, ‘stroke-linejoin’, ‘stroke-miterlimit’, ‘stroke-opacity’, ‘stroke-width’, ‘text-anchor’, ‘text-decoration’, ‘text-rendering’, ‘unicode-bidi’, ‘visibility’, ‘word-spacing’, ‘writing-mode’
  • ‘class’

Attribute definitions:

domain = "<min-value> : <max-value>"
TBS. If the attribute is not specified, the effect is as a value of '-5:5' were specified.
samples = "<integer>"
TBS. If the attribute is not specified, the effect is as a value of '25' were specified.
variable = "<string>"
Sets the macro whose value is set to the different values when the 〈coordinate expression〉, as defined by the ‘function’ attribute, is evaluated.
The #-character is optional when specifying the variable, but it is mandatory when using the variable.
If the attribute is not specified, the effect is as a value of '#x' were specified.
connected = "sharp|smooth"
Indicates how the coordinates produced by the ‘function’ attribute should be connected.
'sharp' indicates that coordinates are connected by issuing line-to commands.
'smooth' indicates that coordinates are connected by issuing curve-to commands.
The control point needed for the shorthand/smooth curveto command is taken by evaluating the coordinate expression at the value for #x, as defined by the ‘variable’ attribute, between the original values, as determined by the ‘domain’ and ‘samples’ attributes.
If the attribute is not specified, the effect is as a value of 'sharp' were specified.
function = "plot data"
The definition of the coordinate expression. See Plot data.
